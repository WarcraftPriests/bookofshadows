I wanted to share a quick update on the new **#tc-shadow** thread:
- https://discord.com/channels/118456785047781384/1482545398504489072

Theorycrafting work for Shadow Priest keeps getting harder every patch cycle. There are more game updates, more tuning passes, more interactions, and more edge cases to test than ever before.

<!-- truncate -->

## Why this thread exists
The goal is simple: do more of this work in the open and make it easier for people to jump in.

A lot of people assume most TC work happens in private channels. The truth is, for sims and APL work it has mostly been a 1-2 person effort for a long time. There has not been some giant private pipeline hiding all the work.

## What has become hard to keep up with
The ongoing load is a lot for one person to handle consistently:
- Tracking and testing APL changes
- Following game changes and updates
- Running and validating the actual sims
- Re-checking results after new tuning and build changes

Each of these steps takes time on its own, and they all need to happen together to keep recommendations accurate.

## What help looks like
If you want to contribute, this new thread is the place to do it.

Useful help can include:
- Reviewing APL diffs and logic changes
- Testing candidate APL edits across different scenarios
- Validating behavior after SimC/module updates
- Sharing reproducible sim setups and results
- Flagging suspicious output or regressions early

## The ask
If you care about Shadow Priest theorycrafting, please join the thread and help us keep this sustainable.

The hope is that the new #tc-shadow thread makes it easier to spread the workload, keep everything transparent, and ship better answers faster.
